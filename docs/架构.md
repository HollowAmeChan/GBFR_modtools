# C++ 架构

## 目标

GBFR Modtools 是 Windows 专用的 C++20 桌面编辑器。UI 使用 Dear ImGui Docking，窗口与输入使用 Win32，预览使用 Direct3D 11。GBFR 自定义格式由独立格式层解析，不经 Assimp 转换。

旧 PowerShell/WinForms 构建器已删除。PowerShell 只作为编辑器内部调用的工作区生成后端，不是第二套 UI 或构建系统。

## Target 边界

```text
GBFRModtools -> gbfr_render  -> gbfr_core
             -> gbfr_formats -> gbfr_core
gbfr_tests   -> gbfr_render + gbfr_formats + gbfr_core
```

- `gbfr_core`：工作区读取、路径约束、SHA-256、修改检测、原子写入与模型构建/恢复。
- `gbfr_formats`：`.minfo/.mmesh/.skeleton/.mmat.json/.mot/.sop` 与 cloth XML 的解析。
- `gbfr_render`：D3D11 设备、DDS 资源、GPU 蒙皮、材质 pass、骨架和 cloth/collision debug draw。
- `GBFRModtools`：Win32 生命周期、ImGui 面板、选择状态和用户命令。

`gbfr_core` 和 `gbfr_formats` 不引用 ImGui 或 D3D11 UI 类型。文件格式解码器可独立测试，动画解码器不依赖渲染器。

## 目录

```text
GBFR_modtools/
  README.md                 用户入口
  build.bat                唯一双击入口
  cmake/                   CMake 项目与 presets
  scripts/
    bootstrap_tools.ps1    固定版本运行工具安装
    workspace/             编辑器内部的工作区生成后端
  src/                     C++ target 实现
  include/gbfr/            跨 target 公开头
  assets/shaders/          HLSL
  tests/                   格式、工作区与渲染回归
  docs/                    用户、开发与格式文档
  _lib/                    schema、名称映射和自动安装的 tools
  .deps/                   可重建的下载/C++ 依赖缓存
  out/                     可重建的编译输出
  explore_output/          未指定输出目录时使用的默认角色工作区
```

根目录不放程序内部 PS1、旧兼容入口或研究文档。CMake 也由 `build.bat` 代理调用 `cmake/`，使用者只需识别一个启动文件。

## 工作区语义

```text
<workspace>/
  manifest.md
  workspace.json
  source/data/              原始恢复基线，不编辑
  unpack/data/              完整可编辑中间态
  build/data/               最终 Mod 输出
```

`workspace.json` Version 1 保存相对路径、基线 SHA-256 和输出映射。`Workspace::resolve` 拒绝逃出工作区根目录的路径。恢复会验证 `source` 哈希，写入 `build` 使用同目录临时文件和替换，避免中途失败留下半个输出。

工作区生成保留完整 `unpack`，不以减少文件数为目标。重构的是代码职责，不是 `source -> unpack -> build` 流程。

## 预览流程

1. `.minfo` 提供顶点/索引数量、chunk、材质索引和包围信息。
2. 选中的 `lod#`/`shadowlod#` `.mmesh` 依照 minfo 对应 LOD 记录解析顶点流、4/8 权重、UV0/UV1、顶点色和索引；选择 minfo 或 skeleton 时回退到 `lod0`。
3. `.skeleton` 提供层级、静止 TRS 和骨骼名编码。
4. `vars/0.mmat.json` 绑定默认配色材质与贴图。
5. MOT 在 60 FPS 帧空间采样本地 TRS，之后 SOP 派生 deform/corrective 骨。
6. CPU 生成层级姿态和蒙皮矩阵，GPU 顶点着色器完成顶点与法线蒙皮。
7. CLP/CLH 调试几何使用动画后骨骼世界变换；`seq_edit_cloth` 时间轴与当前 MOT 同名匹配。

DirectX 贴图在预览时统一处理 V 翻转。面部材质把主表面/口腔 alpha clip 与眉毛睫毛覆盖 blend 分开，骨架和碰撞线在材质 pass 后绘制。

| 功能 | 状态 |
|---|---|
| 工作区创建、路径约束、SHA-256 修改检测 | C++ 原生 + 内部解码后端 |
| minfo/skeleton/mmesh 恢复和原样写入 `build` | C++ 原生 |

| mmat JSON 编码 | 尚未实现 |
| cloth XML 编码为 BXM | 尚未实现 |

界面不保留旧构建器入口。未实现的类型显示为仅查看/中间态，不生成看似成功的不完整 Mod。
