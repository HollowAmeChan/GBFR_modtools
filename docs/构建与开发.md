# 构建与开发

## 环境要求

- Windows 10/11 x64。
- Visual Studio 2022，安装“使用 C++ 的桌面开发”工作负载，包含 MSVC x64、Windows SDK、CMake 和 Ninja。
- Git for Windows，用于 CMake 首次获取 C++ 依赖。
- Windows PowerShell 5.1，系统自带；无需 Python。

不需要手工设置 `PATH`。`build.bat` 通过 `vswhere.exe` 定位 Visual Studio，优先使用 VS 自带的 CMake 和 Ninja。

## 首次启动

```bat
git clone https://github.com/HollowAmeChan/GBFR_modtools.git
cd GBFR_modtools
build.bat
```

无参数运行会按顺序完成：

1. 下载并校验工作区解码工具。
2. 自动定位 Visual Studio、MSVC、CMake、Ninja 和 Git。
3. 使用 CMake FetchContent 将固定版本的 C++ 依赖缓存到 `.deps/`。
4. 增量编译 `RelWithDebInfo`。
5. 启动 `out/bin/RelWithDebInfo/GBFRModtools.exe`。

之后构建会重用 `.deps/` 和 `_lib/tools/`，没有版本变化时不再下载。

## 命令

```bat
build.bat                    构建 RelWithDebInfo 并启动
build.bat run                同上
build.bat test               构建并运行全部测试
build.bat Debug              只构建 Debug
build.bat Release            只构建 Release
build.bat RelWithDebInfo     只构建带调试信息的优化版
build.bat tools              只安装/验证工作区解码工具，不需要 VS
build.bat clean              只删除 out/，保留依赖和工具缓存
```

中文或较长的仓库路径通过临时 `subst` 盘符隔离。脚本会在 `R:` 到 `J:` 之间选择空闲盘符，构建后立即解除，不再依赖固定 `R:`。

## 自动依赖

### 工作区解码工具

`scripts/bootstrap_tools.ps1` 从官方 release 下载到 `.deps/downloads/`，校验下载包和最终 EXE 的 SHA-256，再安装到 `_lib/tools/`。

| 工具 | 固定版本 | 作用 |
|---|---|---|
| [GBFRDataTools](https://github.com/Nenkai/GBFRDataTools) | 2.0.0 | BXM 解码为 XML |
| [FlatBuffers](https://github.com/google/flatbuffers) `flatc` | 25.12.19 | mmat 解码为 JSON |
| [GraniteTextureReader](https://github.com/Nenkai/GraniteTextureReader) | 1.1.5 | GTS/GTP 图层提取 |
| [DirectXTex](https://github.com/microsoft/DirectXTex) `texconv` | may2026 | TGA 转 BC7/BC5 DDS |

这些二进制不提交到 Git。版本、URL 和哈希的单一信息源是 `scripts/bootstrap_tools.ps1`。

### C++ 库

CMake 自动获取 Dear ImGui Docking、nlohmann/json 和 pugixml；可选 Assimp 默认关闭。Git 依赖使用固定 commit，JSON 发布包使用固定 SHA-256。缓存完成后 CMake 使用 `FETCHCONTENT_UPDATES_DISCONNECTED`，日常增量构建不会追踪远程分支。

## 新用户完整流程

1. clone 仓库并运行一次 `build.bat tools`，取得已校验的 GBFRDataTools 2.0。
2. 用下面的命令预先解包游戏 `data.i`；`-o` 指向空间充足的独立目录，最终应能看到 `<输出目录>/data/model/` 与 `<输出目录>/data/model_streaming/`。

```bat
_lib\tools\GBFRDataTools\GBFRDataTools.exe extract-all ^
  -i "D:\Steam\steamapps\common\Granblue Fantasy Relink\data.i" ^
  -o "D:\GBFR_extracted"
```

3. 运行 `build.bat` 编译并启动编辑器。
4. 在编辑器开始页选择 `<输出目录>/data/model/<类型>/<角色>/<角色>.minfo`。
5. 等待 `explore_output/workspace.json` 生成并自动打开。
6. 编辑 `explore_output/unpack/data/`，在编辑器中预览和检查。
7. 模型文件可使用“从 unpack 复制到 build”生成 Mod 目录。尚未原生编码的贴图/mmat/cloth 只能检查中间态，不会被假装成可构建。

重复解包并希望覆盖旧文件时，再给 `extract-all` 添加 `--overwrite`。完整解包会占用较多磁盘空间；编辑器不会直接读取压缩包中的资源，因此不能跳过这一步。

## 排错

- `vswhere.exe was not found`：通过 Visual Studio Installer 安装 Visual Studio 2022 或 Build Tools 2022。
- `MSVC x64 workload was not found`：在 Visual Studio Installer 中勾选“使用 C++ 的桌面开发”。
- `Git was not found`：安装 Git for Windows，重开终端。
- 工具下载失败：检查 GitHub 访问、代理或防火墙，然后单独执行 `build.bat tools`。不会使用未通过哈希校验的半成品。
- CMake 依赖获取失败：确认 Git 可访问 GitHub；修复网络后直接重试，已完成的缓存会保留。
- 需要彻底重新编译：先 `build.bat clean`；不要手动删除 `_lib/tools/` 或 `.deps/`，除非正在排查依赖损坏。
