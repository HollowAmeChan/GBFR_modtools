# Blender 导出对照与排错

本项目使用官方 `GBFR Blender Tools v2.0.0` 作为模型导入/导出的格式对照，固定副本位于 `_references/GBFRBlenderTools-v2.0.0`。官方版本的导出器会生成同一套 `.minfo`、`.skeleton` 和 `model_streaming/lod#/模型.mmesh`，并要求根对象、LOD、Mesh 的层级保持一致。

## 已确认的格式行为

- 本地 `gbfr_model_export_v2.py` 的核心 mmesh 写入布局与官方 v2.0.0 一致：位置/法线/切线/UV0、权重、颜色、UV1、索引缓冲按 `.minfo` 中的顺序写入。
- Blender Z-up 与游戏 Y-up 的转换由导出器统一处理；不要在导出前额外镜像模型或手动交换左右骨骼。
- 骨骼权重索引必须与同一个 `.skeleton` 的骨骼顺序一致，单个顶点最多 8 个权重。
- 权重缓冲是“每个 LOD 一组固定步长”，不能在同一个 LOD 的不同 Mesh 之间混用 4 槽和 8 槽。当前插件已统一按该 LOD 的最大权重数写满 4/8 槽；这是官方旧实现未覆盖的边界情况，也是权重碎裂和游戏崩溃的常见原因。
- 某些原始模型的 UV1/颜色缓冲短于顶点数。它们表示部分附加数据，不应因为缺少尾部数据而越界；导入器会为缺失元素使用 `(0, 0)` 或白色默认值。

## 导出前检查

1. 根对象必须是带骨骼的 Armature（或无骨骼模型的 Empty），子级为 `lod0` 至 `lod4`，LOD 子级为 Mesh。
2. Mesh 的顶点组名称必须能映射到根 Armature 的骨骼；不要在导出前删除实际被权重引用的骨骼。
3. 每个顶点最多 8 个有效权重，并且权重总和应为 1；无权重顶点可能导致游戏中的模型爆炸或加载失败。
4. 模型只使用最多两个 UV 图层；需要拆分 UV 岛时先使用插件的 “Split Mesh along UVs”。
5. 导出后确认同一模型的 `.minfo`、`.skeleton` 与所有 `model_streaming/lod#/*.mmesh` 一起覆盖到 `unpack`，不要只替换单个 `.mmesh`。

## 当前验证

使用 `解包留档/奶刀/source` 的 `pl1400` 做无修改往返时，四个 LOD 的顶点数、索引数、缓冲区偏移/长度均保持一致；这验证了坐标转换和基础写入布局。若导出的模型在外部预览器中左右镜像，先检查 Blender 场景是否存在额外的负 X 缩放或左右骨骼重命名，再检查预览器的坐标显示；不要据此修改 `.mmesh` 的顶点顺序。

官方 v2.0.0 的发布说明记录了完整导入/导出重构、最多 8 权重及面部拉伸修复：[官方发布页](https://github.com/WistfulHopes/GBFRBlenderTools/releases/tag/v2.0.0)。
