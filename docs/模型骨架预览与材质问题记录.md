# 模型骨架、预览器与材质问题记录

这份文档记录当前奶刀模型替换过程中已经观察到的游戏内差异、可复现结论和后续验证计划。文中明确标注为“假设”的内容仍需要通过预览器、资源对比或游戏帧捕获确认。

## 已确认的骨架结论

- Blender 中静态网格看起来对齐，不代表游戏中的最终位置一致。游戏还会应用骨架初始变换、MOT 动画和其他约束，因此身体、头部在游戏里可能出现脖子断开、头部下插或脚部悬空。
- 当前样本确认 `_000`（hips）是位移锚点。将它吸附或移动到另一套骨架后，身体在游戏中会产生整体高度/位置偏移；把 `_000` 恢复到 source 位置后，身体漂移问题消失。
- 当前样本进一步确认 `_005`（head/neck）也不能移动。它和 `_000` 具有相同的位移骨语义，头部替换时必须保留 source 的静止变换，否则会在游戏动画或运行时约束下造成脖子接不上、头部插入身体等问题。
- `_000` 和 `_005` 的位置不能因为“视觉上重叠”就重新吸附。模型替换应当调整网格、权重和普通子骨骼，保留这两个锚点骨骼的原名、父级和 rest transform。
- 不能因为旧头发或其他部件暂时不用就删除旧骨骼。cloth、SOP、MOT 和权重可能按原骨名或固定索引引用它们；完整模型导出仍应保留 source 骨架，再追加需要的新骨。

## 尚待确认的运行时因素

目前最合理的解释是：动画会给 `_000` 写入角色位移，`_005` 则可能承担头部链的相同语义或被独立的头部动画/约束驱动。需要在预览器中分别观察它们的 rest transform 和动画帧位移，才能区分以下情况：

1. 骨架的 source 初始变换已经不一致。
2. MOT 对 `_000` 或 `_005` 写入了非零平移。
3. 头部使用了独立动画、SOP 或 cloth 约束，导致静态对齐后运行时再次偏移。
4. 网格顶点/权重正确，但模型对象的根变换或分段对象变换没有保持 source 值。

## 预览器待办

这些功能用于诊断，不应在预览过程中修改导出数据：

- 在 ImGui 预览模式显示明确的地板/网格平面，用于观察脚部是否悬空、根骨高度和整体位移。
- 支持同时预览多个选中的 mesh/model 资源。例如身体和头部应能在同一个场景中静态叠加，而不是后选对象替换前一个对象。
- 多选预览播放动画时共享时间轴、播放/暂停和速度，使所有模型段同步求值。
- 增加“联动头部锚点”开关，默认开启：多选身体和头部时，将头部 `_005` 在预览求值中对齐到身体对应的 source 位置。该开关只影响预览诊断，不应改写 Blender 或导出文件。
- 在预览器中显示 `_000`、`_005` 的 source/rest transform、当前帧 transform 和平移差值，直接判断是哪一级产生了漂移。
- 多段模型预览应明确显示使用的 `source` 动画、SOP 和 cloth 配置，避免误把不同工作区或 `build` 文件混进诊断结果。

## 材质与贴图待办

- 当前部分材质在游戏中显示过黑。现有 mmat 解码工具对新 DLC 增加的材质字段覆盖不完整，未知字段可能被丢弃或被错误地映射到默认值；需要以新 DLC 的 mmat 样本和 schema/字段表补齐解码、保留和再编码路径。
- 单面材质的透明效果还没有在编辑器中提供独立控制。需要明确区分背面剔除、双面渲染、alpha blend 和 alpha clip，并在材质编辑界面提供可保存到 mmat 的设置。
- 贴图封回与预览的颜色空间、法线格式和 Granite/普通 `.texture` 路径需要继续按材质类型核对；“预览正常”不能单独证明游戏内材质状态正确。

## 平整表面毛躁/噪波

观察到模型平整表面进入游戏后，在某个方向出现明显毛躁噪波；沿着模型与世界原点连线的方向不明显，接近垂直方向时更明显。目前不能把它归因于单一问题，优先排查：

- 法线、切线或 bitangent 的方向/手性在导出后与 source 不一致。
- 法线或切线缓冲的量化、归一化或顶点拆分造成平面上的微小不连续。
- mmat 中的法线贴图、roughness、镜面参数或颜色空间解释不正确。
- 蒙皮后法线变换、对象根变换或 LOD 切换导致方向相关的闪烁。
- 游戏的镜面高光、TAA 或阴影采样放大了上述微小误差。

建议按以下顺序验证：

1. 对同一平面区域逐顶点比较 source 与导出 mesh 的 position、normal、tangent 和 UV。
2. 在游戏中临时关闭法线贴图和镜面/roughness，确认噪波是否消失。
3. 改变模型朝向、相机方向和世界位置，判断噪波是随模型、光照还是世界原点方向变化。
4. 在预览器增加 normal/tangent debug 显示；必要时使用游戏帧捕获确认最终 GPU 输入。

## 当前操作原则

- 替换身体、头部或其他分段时，保留 source 的 `_000`、`_005` 以及全部原骨架层级；不要移动这两个位移锚点，也不要为了清理旧部件删除可能被 cloth/SOP/MOT 引用的旧骨。
- 先在多 mesh、多骨架联动预览中确认静态接缝和动画接缝，再将 `.minfo`、`.skeleton` 与所有相关 `.mmesh` 作为一组构建到 `build`。
- 任何尚未证实的材质或噪波原因都应记录为假设，并保留 source 资源用于逐项对比。
